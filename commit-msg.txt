Favicon + og:image refresh for social sharing

Inspect state on entry:
- favicon.svg existed (clean sprout on dark with emerald stroke) — kept
- og-image.svg had great design but stale text ("5 mutations") —
  was from Phase 2, before Starstruck + Bloodlit were added in Phase 11
- og-image.png was rasterized from the stale SVG (still 1200x630
  but shows the wrong mutation count)
- manifest.webmanifest PWA shortcuts still used the obsolete
  hash-router format (#/crops, #/pets, #/calculator, #/compare)

Changes:

public/og-image.svg
  Stats updated to current state (Phase 12 final):
  - "5 mutations" -> "6 mutations"
  - added a second subtitle line: "+ 27 gears · 3 eggs · 4 seed
    packs · 12 pets" so the new categories from Phase 12 are
    represented in the share preview

public/og-image.png
  Regenerated from the updated SVG using sharp (now a devDep).
  Output: 1200x630, 114,320 bytes. density 144 to keep text crisp,
  compression level 9 for the smallest possible file.

public/manifest.webmanifest
  PWA shortcut URLs: "#/crops" -> "./crops" (and same for pets,
  calculator, compare). The browser-router switch in Phase 14
  broke these for any user with the PWA installed.
  Pet description updated to "all 12 pets" (Phase 12b).

public/favicon.svg
  Left as-is. 577 bytes, simple sprout, renders at 16x16 in
  modern browsers. Good enough.

scripts/regen-og-image.cjs (new)
  One-off Node script that reads public/og-image.svg and writes
  public/og-image.png. Run with `node scripts/regen-og-image.cjs`
  after editing the SVG. Kept in the repo (not gitignored) so
  future contributors can regenerate the PNG.

devDependency added: sharp
  Required for SVG -> PNG. Adds ~30 MB to node_modules. Kept
  in devDeps (not bundled) so it doesn't bloat the production
  build. Worth it for the og-image regen.

Meta tags (index.html)
  Already comprehensive (og:image:width/height/alt, twitter
  card=summary_large_image). No changes needed.

Future maintenance:
- Edit og-image.svg for any visual change to the share preview
- Run `node scripts/regen-og-image.cjs` to regenerate the PNG
- Sharp is a devDep so it ships with the project; can be removed
  if the OG image becomes stable and the regen script is unused
