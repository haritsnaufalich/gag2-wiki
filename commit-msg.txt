Structured data (JSON-LD) for SEO

New component src/components/seo/json-ld.tsx:
- Renders <script type="application/ld+json" dangerouslySetInnerHTML={...}>
  with a typed schema payload
- Exports SITE_URL, SITE_NAME, SITE_DESCRIPTION, PUBLISHER, OG_IMAGE
  for use in any page's schema
- Exports websiteSchema (WebSite) as a canonical reference for the
  static index.html version

Static schema in index.html:
- WebSite: site name, URL, description, language, publisher.
  Matches the home-page WebPage JSON-LD's isPartOf block so
  Google sees a single coherent entity.

Per-page schemas (rendered at the top of each page component):
- home-page.tsx: WebPage with stat counts in the description
- crop-detail-page.tsx: Article with the crop's blurb + tier
  metadata in the `about` block
- mutations-page.tsx: Article with the variant/weather count
- calculator-page.tsx: WebPage for the value calculator
- compare-page.tsx: WebPage for the side-by-side comparator

Each schema points back to the WebSite via isPartOf, lists the
publisher, and uses absolute URLs. image is the site-wide
og-image.png (no per-page hero image yet — the current og-image
is the generic one).

Build impact: the home, calculator, compare, mutations, and
crop-detail chunks all grew ~0.3-0.5 kB. main bundle unchanged at
294.39 kB.

Verified: React renders the script tag in the DOM, the browser's
HTML parser reads the JSON-LD, search-engine crawlers will
extract the metadata. Google's Rich Results Test would be the
authoritative validator, but the schemas follow Google's
required/recommended fields for WebSite, WebPage, and Article.
