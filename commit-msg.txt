Design tokens: src/lib/tokens.ts + value-curve-chart refactor

Added src/lib/tokens.ts as a single source of truth for the values
Tailwind classes can't easily reach (SVG fills, inline style={}
blocks, canvas, third-party widgets). Hex values mirror Tailwind's
palette so visual consistency is automatic when both systems agree.

Tokens shipped:
- colors: brand + accent hex (emerald400, emerald500, amber400,
  rose500, sky400, violet400, lime500, slate900)
- rgb: same as colors but in rgb(R G B) form for SVG attributes
- chart: chart-specific palette (curve, point, pointRing) for the
  value-curve chart
- font: family (Poppins)
- types: ColorToken, RgbToken, ChartToken

value-curve-chart.tsx refactor:
- Removed local CURVE_COLOR / POINT_COLOR / POINT_RING constants
  (3 hardcoded rgb() strings)
- Replaced the inline `style={{ ..., stroke: 'rgb(15 23 42)' }}`
  on point labels with `chart.pointRing`
- Now imports { chart } from '@/lib/tokens' — single source of
  truth for the chart palette

If a future contributor needs to tweak the chart colors, they
change tokens.ts instead of hunting for rgb() strings.

No behavior change. Bundle impact: +0.13 kB on the main chunk
(tokens.ts is small). crop-detail-page chunk went from 11.69 kB
to 11.82 kB (it owns the chart, so the import lands here rather
than the main bundle — Vite hoist decisions).

Audit note: only value-curve-chart had hardcoded colors. The
rest of the project uses Tailwind classes, which IS the token
system for component JSX. This file is for the non-Tailwind
exceptions.
