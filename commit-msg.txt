UI revamp: simplify home page (less clutter, better UX)

The home page was over-stuffed with 6 competing sections:
1. Hero with 7-stat grid
2. 9 quick-link cards (3x3)
3. 5 pack-seeds cards
4. 4 featured-crop cards
5. 8-tier ladder
6. Active code banner

Per the ui-ux-pro-max principle "one dominant element per
viewport", the page now has 4 focused sections:

1. Hero (cleaner)
   - Live badge with compact stats line
     ("36 crops · 4 mutations · 8 tiers")
   - H1 with gradient accent
   - Tight description (no comma list, prose-style)
   - 2 CTAs (Browse Crops primary, Calculator secondary)
   - NO more 7-stat card grid (was visual noise)

2. 4 quick-link cards (down from 9)
   - Crops, Calculator, Mutations, Pets
   - Each with icon, title, 1-line desc, "Open" arrow
   - Hover ring accent per card (emerald/amber/violet/rose)
   - Focus-visible ring (keyboard a11y)
   - sr-only section heading for screen readers

3. 4 featured-crop cards (compact, was 4 + tier ladder)
   - Tier order: lowest -> highest rarity
   - Reuses existing TierBadge + CropCard pattern
   - "View all 36 ->" link
   - Tier ladder REMOVED (covered by /crops page)

4. Active code banner (kept)
   - Centered card with gradient amber background
   - Code (TEAMGREENBEAN) on the right, monospace emerald

Removed (in favor of /crops, /seed-packs, /pets pages):
- 7-stat grid in hero
- 5 of 9 quick-link cards (kept only the 4 most-used)
- Pack-seeds strip (5 pack-exclusive crops)
- Tier ladder (8 tier dots in a row)

Net change: 6 sections -> 4, 9 feature cards -> 4, 5 pack
cards removed, 1 tier ladder removed. Hero 7-stat grid
removed. ~120 lines lighter overall.

Bundle: main 298.67 kB -> 298.62 kB (no measurable diff;
removed sections were small, new Hero copy slightly larger).
Build clean, 0 TS errors.

Live audit (pending deploy):
- /gag2-wiki/ should still resolve to 200 with the
  revamped hero + 4 quick links + 4 featured crops + code
  banner
- All other routes unchanged

No new dependencies. No new files (single-file change
in src/pages/home-page.tsx).
