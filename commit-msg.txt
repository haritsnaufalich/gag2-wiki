Hash-router -> browser-router migration shim in index.html + 404.html

External links and bookmarks that point at the old hash format
(/#/crops, /#/crops/bamboo, /#/calculator?weight=1, ...) silently
landed on the home page after the browser-router switch, because
createBrowserRouter reads window.location.pathname (which never
includes the hash) and not the hash.

Add a 14-line IIFE to both index.html and 404.html that runs
before the React bundle. It detects the old '#/' prefix and
rewrites the URL via history.replaceState (so the back button
stays clean). Search params are preserved.

- Strict prefix check: only matches '#/' (not '#section' anchors)
- IIFE — no global pollution
- replaceState — no extra history entry
- runs synchronously, before the module script tag

This can be removed in a few months once external references
have migrated. Comments at the top of each block explain why.
